## 一、系统总览

### 1.1 产品定位

专业的中医执业医师考试在线刷题平台，核心目标为帮助用户高效通过中医执业医师资格考试。

### 1.2 用户群体

| 角色 | 说明 |
|------|------|
| 中医执业医师考生 | 备考执业医师资格证 |
| 中医助理医师考生 | 备考助理医师资格证 |
| 中医学本科生 | 在校学习辅助 |
| 中医学研究生 | 深造备考 |
| 规培医生 | 规范化培训期间备考 |
| 编辑 | 后台管理题库内容 |
| 审核员 | 审核题目内容质量 |
| 超级管理员 | 系统全面管理 |

### 1.3 核心原则

- 优先学习效率，而非社交功能
- 页面简洁，减少视觉噪音
- 所有功能围绕"刷题—解析—复习—通过考试"展开
- 持续更新真题和考纲
- 支持 PC 和移动端自适应

## 三、题库系统

### 3.1 题型支持

| 题型 | 代码 | 说明 |
|------|------|------|
| 单项选择题 | single | 四选一 |
| 多项选择题 | multiple | 至少两个正确答案 |
| A1 型题 | a1 | 单句型最佳选择题 |
| A2 型题 | a2 | 病历摘要型最佳选择题 |
| B1 型题 | b1 | 共用备选答案题（配伍题）|
| 案例分析题 | case | 基于临床案例的多问答题 |

### 3.2 题目属性

| 属性 | 类型 | 说明 |
|------|------|------|
| id | string | 题目唯一 ID |
| type | enum | 题型枚举 |
| year | number | 真题年份 |
| examCategory | string | 考试类别：执业医师 / 助理医师 |
| subject | string | 考试科目 |
| knowledgePoint | string | 知识点 |
| difficulty | 1-5 | 难度等级 |
| source | string | 来源 |
| tags | string[] | 标签数组 |
| content | string | 题目正文 |
| options | Option[] | 选项列表 |
| answer | string[] | 正确答案 |
| explanation | string | 答案解析 |
| clinicalThinking | string | 临床思路 |
| keyPoints | string[] | 考点总结 |
| commonMistakes | string[] | 易错点提醒 |
| relatedQuestions | string[] | 关联题目 ID |
| relatedKnowledge | string[] | 关联知识点 |
| status | enum | 审核状态 |
| version | number | 版本号 |

### 3.3 筛选维度

| 筛选 | 说明 |
|------|------|
| 年份 | 按真题年份筛选 |
| 科目 | 按考试科目 |
| 知识点 | 按具体知识点 |
| 难度 | 1-5 级 |
| 错题 | 仅显示做错的题目 |
| 收藏题 | 仅显示收藏的题目 |
| 未做题 | 仅显示从未作答的题目 |

### 3.4 批量导入

**支持格式：** Excel (.xlsx)、CSV、JSON

**导入流程：** 上传文件 → 格式校验 → 字段映射 → 预览确认 → 执行导入 → 结果报告

### 3.5 版本管理

- 每次编辑题目生成新版本
- 保留历史版本记录
- 支持版本对比（diff 视图）
- 支持回滚到历史版本

### 3.6 审核流程

编辑提交 → 审核员审核 → 通过/驳回 → 编辑修改重提

**审核状态：** draft → pending → approved / rejected

## 四、刷题系统

### 4.1 刷题模式

| 模式 | 说明 |
|------|------|
| 顺序练习 | 按科目/章节顺序逐题练习 |
| 随机练习 | 从题库中随机抽取题目 |
| 模拟考试 | 限时按真实考试标准组卷 |
| 专项训练 | 针对某科目/知识点集中练习 |
| 历年真题 | 按年份筛选完整真题卷 |
| 每日一练 | 每日固定题量推送 |

### 4.2 刷题流程

展示题目 → 用户答题 → 提交答案 → 展示解析 → 下一题

### 4.3 解析内容

| 内容 | 说明 |
|------|------|
| 正确答案 | 高亮标出正确选项 |
| 答案解析 | 详细推理过程 |
| 临床思路 | 结合临床场景的解题思路 |
| 考点总结 | 核心考点 |
| 易错点提醒 | 常见错误选项和陷阱 |
| 关联知识点 | 跳转链接 |
| 相关题目推荐 | 同类题型推荐 |

### 4.4 用户交互

- 收藏题目
- 标记重点
- 添加笔记
- 分享题目

## 五、错题系统

### 5.1 自动记录

- 题目 ID、错误选项、正确答案
- 答错次数（累加）
- 最近答错时间
- 错误原因（用户可选填）

### 5.2 筛选维度

| 筛选 | 说明 |
|------|------|
| 科目 | 按考试科目 |
| 知识点 | 按具体知识点 |
| 时间范围 | 最近 7 天 / 30 天 / 自定义 |
| 错误次数 | 错 1 次 / 2 次 / 3 次以上 |

### 5.3 掌握状态

| 状态 | 说明 |
|------|------|
| 未掌握 | 首次进入或连续答错 |
| 复习中 | 已重新练习 1-2 次 |
| 已掌握 | 连续答对 3 次 |

### 5.4 其他功能

- 重新练习错题（随机/顺序）
- 智能去重
- 错题导出（PDF / Excel）
- 基于遗忘曲线的复习提醒

## 十二、API 接口设计

### 示例接口

`
GET    /api/v1/questions           获取题目列表
GET    /api/v1/questions/:id       获取题目详情
POST   /api/v1/questions           新增题目
POST   /api/v1/auth/login          用户登录
GET    /api/v1/answers             答题记录
POST   /api/v1/answers             提交答案
GET    /api/v1/wrong-record        错题记录
GET    /api/v1/favorites           收藏列表
GET    /api/v1/stats/dashboard     首页统计
GET    /api/v1/stats/analysis      考试分析
`

### 统一响应格式

`json
{
  "code": 200,
  "message": "success",
  "data": {},
  "meta": { "total": 100, "page": 1, "pageSize": 20 }
}
`
