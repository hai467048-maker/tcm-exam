## 一、设计理念

### 1.1 设计价值观

| 原则 | 说明 |
|------|------|
| 留白 | 内容呼吸感，每个元素都有足够的空间 |
| 克制 | 只保留必要的元素，移除一切装饰性的视觉噪音 |
| 层级 | 通过字号、字重、间距建立清晰的信息层级 |
| 秩序 | 网格化的布局，对齐是一切的基础 |
| 质感 | 微妙的阴影、柔和的圆角、克制的色彩过渡 |

### 1.2 东方美学表达

通过以下方式传达东方美学：
- 非对称的平衡（借鉴宋代山水画的构图）
- 大量留白（类似书法中的"计白当黑"）
- 克制的色彩（类似水墨画的"墨分五色"）
- 垂直方向的节奏感（向下滚动的韵律）
- 细微的材质感（宣纸般的米白底色）

## 二、设计规范

### 2.1 色彩系统

#### 主色调

```
米白 (背景)      #F5F0EB  — 宣纸色，温暖克制的主背景
墨绿 (品牌色)    #2D5016  — 定海神针，用于重要操作和标题
浅灰 (辅助色)    #E8E5E0  — 分割线、卡片背景、次要信息

辅助色:
深墨绿          #1A3308  — 更深沉的强调色
中墨绿          #4A7C23  — 悬停状态
浅墨绿          #7FA856  — 次要强调
极浅绿          #EDF3E8  — 正确/正确反馈背景
暖灰            #D1CDC7  — 禁用状态、分割线
浅暖灰          #F0EDE8  — 卡片背景白、hover
白              #FFFFFF  — 纯白卡片
```

#### 语义色

```
正确 (绿)       #3B7A1E  / 背景 #EDF3E8
错误 (红)       #B33A3A  / 背景 #FDF0F0
警告 (橙)       #C4701A  / 背景 #FFF5EB
信息 (蓝)       #2B6A9E  / 背景 #EBF3FA
```

#### 文字色

```
标题/正文       #1A1A18  — 接近黑色但略暖
次要文字       #6B6A66  — 说明文字
辅助文字       #9C9A96  — 时间戳、次要信息
占位符         #B8B5B0  — 输入框占位
反色文字       #F5F0EB  — 墨绿背景上的文字
```

### 2.2 字体系统

```
字体栈: "Inter", "Noto Sans SC", -apple-system, "PingFang SC",
         "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif

字号层级:
  xs:   12px (0.75rem)  — 标签、辅助文字
  sm:   14px (0.875rem) — 正文、说明文字
  base: 16px (1rem)     — 默认正文
  lg:   18px (1.125rem) — 小标题、强调
  xl:   20px (1.25rem)  — 卡片标题
  2xl:  24px (1.5rem)   — 页面标题
  3xl:  30px (1.875rem) — 大标题
  4xl:  36px (2.25rem)  — 首页主标题
```

### 2.3 间距系统

```
基于 4px 网格:
  4:  16px   — 卡片内边距
  6:  24px   — 区块间距
  8:  32px   — 大区块间距
  10: 40px   — 页面分区
  12: 48px   — 大分区

页面最大宽度: 1200px
内容区 padding: 24px (移动端) / 40px (桌面端)
```

### 2.4 圆角

```
sm:     4px  — 小元素（标签、徽章）
md:     8px  — 卡片、按钮
lg:     12px — 大卡片、弹窗
xl:     16px — 首页大区块
full: 9999px — 圆形（头像、指示器）
```

### 2.5 阴影

```
sm:   0 1px 2px rgba(26,26,24,0.04)    — 卡片常态
md:   0 2px 8px rgba(26,26,24,0.06)   — 卡片悬停
lg:   0 4px 16px rgba(26,26,24,0.08)  — 弹窗
inner: inset 0 1px 2px ...             — 输入框内阴影
```

### 2.6 过渡动画

```
默认:  200ms ease — 悬停、切换
强调:  300ms ease — 页面切换、弹窗
缓入:  150ms ease-in — 进入
缓出:  250ms ease-out — 离开
```

| 主要按钮 | 提交答案、开始考试 | 墨绿(#2D5016)背景 + 白字 + 8px圆角 + 44px高 |

### 3.2 卡片

```

  hover: md阴影

### 3.3 导航

```

  激活态: 左侧3px墨绿条 + 极浅绿背景

  顶部56px白条 + 汉堡按钮
  侧边栏从左侧滑入(300ms) + 半透明遮罩
```

### 3.4 标签

```

## 五、交互流程

### 刷题核心流程

```
1. 用户点击选项 → 选项高亮（墨绿边框+背景）
2. 点击确认答案
   - 正确: 选项变绿 + ✅ 回答正确
   - 错误: 选项变红（选中）变绿（正确答案）
3. 解析面板向下展开（300ms动画）
4. 用户阅读解析 → 点击下一题
5. 当前题淡出 → 下一题淡入（200ms）
```

### 导航流程

```
桌面: 侧边栏固定可见 → 点击菜单切换页面 → 淡入动画
移动: 汉堡按钮 → 侧边栏从左侧滑入(300ms)
      → 点击菜单项或遮罩关闭 → 页面切换
```
